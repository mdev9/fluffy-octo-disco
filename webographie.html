Outils utilisés

Git et GitHub

Git est un logiciel de gestion de versions décentralisé, cela permet de gérer l'évolution du projet. On peut utiliser GitHub avec pour mieux gérer ses repositories et stocker son code dans le cloud. Il s'utilise en ligne de commande, dans son navigateur ou avec l'application. Voici le repo de ce site web où l'on peut voir toutes les différentes version que j'ai commit et push dessus.

IDE et extensions

J'ai utilisé Visual Studio Code, un éditeur de code extensible, open source, gratuit et très pratique. Il intégre aussi directement Git et GitHub. Voici les extensions que j'ai le plus utilisées:

  • Debugger for firefox: permet de débugger les applications webs en temps réel directement dans Firefox.
  • ESLint : analyse le code et trouve les erreurs dans le javascript. Très customisable pour le faire fonctionner comme on veut. Il nécessite Node.js (ou runtime équivalent) d'être installé sur votre ordinateur.
  • TabNine AI Autocomplete : il permet d'autocompleter le code de façon très efficace car l'IA s'adapte à vos besoins et il vous permet donc de devenir plus productif.
  • Beautify : permet de modifier le style du code dans l'éditeur et de le personaliser selon nos goûts de façon très détaillée. Il permet aussi de formatter la plus part des languages en utilisant alt+shift+F.

Navigateur

Firefox, équipé de ses outils pour développeur, très pratique pour mettre des points de débogage pour le javascript, ou faire des tests dans la console. Une version spéciale pour développeurs existe également incluant les derniers outils en version beta et des fonctionalitées expérimentales, un profil et un chemin séparés pour que vous puissiez facilement l’exécuter en même temps que Firefox.

Site respectant les normes W3C

La norme W3C, qu'est-ce que c'est ?

La norme W3C (World Wide Web Consortium) définit les standards techniques liés au web et surtout détermine les mêmes règles pour tous les développeurs du monde. L’idée est également de favoriser l’accès au web, au plus grand nombre.

Pourquoi développer un site aux normes W3C ?

Il peut s’avérer utile de respecter les normes W3C pour différentes raisons : pour votre référencement naturel, en effet, un site construit selon les normes W3C est plus facilement lisible par les moteurs de recherche. Il sera mieux référencé et votre visibilité sera donc d’autant plus importante.

Vous souhaitez vérifier si votre site respecte les normes W3C ?

L’organisme W3C vous propose un outil pour l’examiner, W3C validator.
